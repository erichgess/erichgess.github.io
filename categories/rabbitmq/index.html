However, there’s still some inconsistency in the design which I want to polish out: To create a consumer on a queue, you call: createQueueConsumer myChannel "myQueue" To create a read function for a queue, you call: let (readFrom,_) = createQueueFuntions myChannel let readFromMyQueue = readFrom "myQueue" To create a write function for a queue, you call: let (_,writeTo) = createQueueFuntions myChannel let writeToMyQueue = writeTo "myQueue" Why have a function which manages both the read AND the writes for a channel?
